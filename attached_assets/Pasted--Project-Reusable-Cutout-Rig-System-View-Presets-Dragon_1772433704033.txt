## Project: Reusable Cutout Rig System + View Presets + DragonBones Export (BITRUVIUS)

### Summary
Build a reusable “custom cutout” rigging layer on top of the existing BITRUVIUS skeleton editor so you can:
- Rig quickly with **bone-attached cutouts** (images + procedural colors/shapes).
- Save **multiple view presets** (front/side/back/3/4) as poses + depth/layering overrides on a **single underlying rig**.
- Export both (a) an **internal Rig Preset** you can reuse across characters and (b) a **DragonBones 5.x** compatible `.zip` package.

---

## Goals / Success Criteria
1. **Fast rigging workflow**
   - Create/edit “cutout slots” attached to bones (parent→child) and assign either:
     - Uploaded image (PNG/SVG), or
     - Procedural colored shape (capsule/rect/circle) as placeholder or final style.
   - Drag-and-drop to assign cutouts to bones/slots with sensible defaults (anchor/scale/rotation).

2. **Multi-view support on one rig**
   - Create “Views” like `Front`, `Side`, `Back`, `3/4`.
   - Each view stores:
     - A **pose snapshot** (joint offsets), and
     - **Depth/layer overrides** (z-order, visibility, optional left/right swap rules).
   - Switching views updates the canvas immediately.

3. **Exports**
   - Export an internal **Rig Preset** (versioned schema) that can be re-imported.
   - Export a **DragonBones 5.x zip**: DragonBones JSON + atlas JSON + atlas PNG (packed) + any referenced images if needed.

---

## Key Product Decisions (Locked)
- Export target: **Both** (internal preset + DragonBones export).
- Views: **Single rig** with **view presets** + **depth layers**, and views can be saved from current pose.
- Cutouts: **Images + colors/shapes**.
- Attachment: **Bone slots** by default (parent→child), with optional joint-attach available later if needed.

## Rendering Changes (Canvas/SVG)
In `/Users/bradleygeiser/Downloads/zipFilezip/client/src/App.tsx`:
1. Replace `jointMasksLayer` + `headMaskLayer` with a single `cutoutsLayer` that:
   - Resolves each `CutoutSlot` attachment to a bone vector:
     - world positions from `getWorldPosition(fromJointId)` and `getWorldPosition(toJointId)`
     - angle from vector
     - length in px
   - Computes final rect/shape bounds based on:
     - `mode`:
       - `cutout`: rigid sizing (asset natural size * scale, or head-relative fallback if unknown)
       - `rubberhose`: height follows bone length * lengthScale; width follows thickness (with optional volume preserve)
       - `roto`: ignores bone angle for rotation (keeps world-aligned unless overridden)
   - Sorts by effective zIndex:
     - base slot `zIndex`, overridden by active view `slotOverrides[slotId].zIndex`
2. Support “depth layer” presets by storing zIndex values per view (override only, no duplication).

## Exports

### A) Internal “Rig Preset” export/import (versioned)
Create `/Users/bradleygeiser/Downloads/zipFilezip/client/src/engine/export/rigPreset.ts`:
- `schema: 'bitruvius-rig-preset@1'`
- Payload includes:
  - joints topology identifier (e.g., `modelId: 'human_v1'`) + any custom joint offsets you want as rest pose
  - `assets` (optionally embedded as data URLs)
  - `cutoutSlots`
  - `views`
- UI: “Export Rig Preset (.json)” + “Import Rig Preset”.

### B) DragonBones 5.x export (.zip)
Create `/Users/bradleygeiser/Downloads/zipFilezip/client/src/engine/export/dragonBones.ts`:
- Add dependency: `jszip`.
- Export flow:
  1. Choose export scope: **Active view only** (v1) with later option “All views as separate armatures”.
  2. Build DragonBones bones:
     - One bone per joint id, parented by joint.parent.
     - Use current pose (after applying active view) for rest transforms.
  3. Build slots/displays from `cutoutSlots`:
     - Each slot attaches to the *fromJointId* bone (or a derived bone); store display transform to align image on the segment.
     - Apply z-order by ordering slots (and/or zOrder fields) based on computed zIndex.
  4. Build texture atlas:
     - Load all used image assets into `<canvas>`.
     - Pack them into a single atlas PNG with a simple shelf packer.
     - For shape assets: rasterize to an image tile first (same pipeline).
     - Write `texture.json` + `texture.png`.
  5. Zip files:
     - `skeleton.json` (DragonBones data)
     - `texture.json`
     - `texture.png`
- UI: “Export DragonBones (.zip)”.
